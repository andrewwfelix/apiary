Structuring your affiliate categories into "clusters" is a great way to build high-intent pages that convert. Given your interest in technical specs (like your Trek bicycle component knowledge) and history (The Odyssey/Carthage), we can create some really unique angles.

Here are the proposed categories for The Apiary Guide:

1. The Library (Educational Resources)
The Beginner’s Shelf: Core "how-to" books (e.g., Beekeeping for Dummies, The Beekeeper’s Handbook).

Historical & Literary Honey: Books on ancient beekeeping (Carthaginian/Greek methods) and nature writing.

Technical Manuals: Advanced queen rearing and hive biology texts.

2. The Wardrobe (Protective Gear)
Armor & Suits: Full ventilated suits vs. jackets.

The Essentials: Gloves (goatskin vs. nitrile) and veils/hats.

Kid-Sized Gear: Family-focused beekeeping apparel.

The Boxes: Deep vs. Medium supers, 8-frame vs. 10-frame configurations.

const https = require('https');
const path = require('path');

// Logic to download image
const downloadImage = (url, slug) => {
  const file = fs.createWriteStream(path.join(__dirname, 'assets/img/products', `${slug}.jpg`));
  https.get(url, (response) => {
    response.pipe(file);
  });
};

// Logic to update JSON
const updateVault = (data) => {
  const vault = JSON.parse(fs.readFileSync('apiary-product-vault.json', 'utf8'));
  vault[data.slug] = {
    asin: data.asin,
    fullName: data.fullName,
    price: data.price,
    url: data.url,
    image: `assets/img/products/${data.slug}.jpg`
  };
  fs.writeFileSync('apiary-product-vault.json', JSON.stringify(vault, null, 2));
};

Replicating a tool like AAWP for your custom static-site setup is a brilliant way to maintain the "Content Factory" momentum without the bloat of WordPress. By creating a custom Affiliate Search \& Curation Dashboard, you move from manual "link fishing" to a structured, data-driven workflow.



Here is a high-level blueprint for building this tool and integrating it into your existing Grok v1.1 pipeline.



1\. The "Affiliate Curator" UI Concept

Since you are a developer, you don't need a full WordPress back-end. You can build a simple local HTML/JS dashboard (running on localhost) that does the following:



Search Bar: You paste an Amazon URL or ASIN.



Scraper Engine: A small Node.js backend uses Puppeteer or a scraping API to "sniff" the product title, primary image, and price.



The "Add to Vault" Button: When clicked, it:



Downloads the high-res product image to your /assets/products/ folder.



Generates the affiliate link using your tag.



Appends a new entry to a product-vault.json file.



2\. The product-vault.json Schema

This file becomes the "Single Source of Truth" that your main build script will reference.



JSON

{

&#x20; "beekeeping-for-dummies-blackiston": {

&#x20;   "fullName": "Beekeeping For Dummies by Howland Blackiston",

&#x20;   "affiliateUrl": "https://amzn.to/example-link",

&#x20;   "imageLocalPath": "/assets/products/beekeeping-dummies.jpg",

&#x20;   "pricePoint": "$18.99",

&#x20;   "lastUpdated": "2026-04-15"

&#x20; }

}

3\. Pipeline Integration (The "Injection" Phase)

In your pipeline-apiary-batch.js script, you would add a helper function to swap placeholders for real data.



Step A: In your manifest or notes, you just use the slug: {{product:beekeeping-for-dummies-blackiston}}.



Step B: During Pass 2 (Haiku), the script looks up the slug in product-vault.json.



Step C: It injects a standardized "Product Box" HTML component into the page:



JavaScript

function injectProductBox(slug) {

&#x20; const data = productVault\[slug];

&#x20; return `

&#x20;   <div class="product-box">

&#x20;     <img src="${data.imageLocalPath}" alt="${data.fullName}">

&#x20;     <h3>${data.fullName}</h3>

&#x20;     <p class="price">${data.pricePoint}</p>

&#x20;     <a href="${data.affiliateUrl}" class="button-accent">View on Amazon</a>

&#x20;   </div>

&#x20; `;

}

4\. Why This Beats WordPress

Performance: There are no database queries or heavy plugins slowing down the site. Your pages remain static and lightning-fast.



Compliance: By downloading images to your own server, you avoid "hotlinking" (which Amazon sometimes dislikes) and ensure your site doesn't break if an image URL changes.



Scalability: Once a product is in the "Vault," you can mention it across 100 different pages by just using its slug. If the price changes, you update it in one JSON file, and a re-build updates the whole site.



5\. Automation Warning: The "API Gate"

Amazon recently retired their old "Product Advertising API" and replaced it with the Creators API.



The Pro Move: If you don't have API access yet, your "Affiliate Curator" dashboard can use a simple Chrome Extension or a Bookmarklet. When you are on an Amazon page, you click the bookmarklet, and it sends the product data to your local Node.js server.

const express = require('express');
const fs = require('fs-extra');
const path = require('path');
const axios = require('axios'); // For downloading images
const app = express();

const cors = require('cors');
app.use(cors()); // Add this before your routes

app.use(express.json());
app.use(express.static('public'));

const VAULT_PATH = path.join(__dirname, 'data/product-vault.json');

app.post('/api/add-product', async (req, res) => {
    const { slug, fullName, url, imageUrl, price, asin } = req.body;
    
    try {
        // 1. Download Image to local storage
        const imgName = `${slug}.jpg`;
        const imgPath = path.join(__dirname, 'public/images', imgName);
        const response = await axios({ url: imageUrl, responseType: 'stream' });
        response.data.pipe(fs.createWriteStream(imgPath));

        // 2. Update the Vault
        const vault = await fs.readJson(VAULT_PATH).catch(() => ({}));
        vault[slug] = {
            asin,
            fullName,
            affiliateUrl: url,
            localImage: `/affiliate-automation/public/images/${imgName}`,
            price,
            lastUpdated: new Date().toISOString()
        };
        
        await fs.writeJson(VAULT_PATH, vault, { spaces: 2 });
        res.json({ success: true, message: `Saved ${slug}` });
    } catch (err) {
        res.status(500).json({ error: err.message });
    }
});

app.listen(3000, () => console.log('Affiliate Service running on http://localhost:3000'));

const vault = JSON.parse(fs.readFileSync('./affiliate-automation/data/product-vault.json'));

function injectAffiliateBoxes(content) {
  // Regex to find [[AFF:product-slug]]
  return content.replace(/\[\[AFF:(.*?)\]\]/g, (match, slug) => {
    const p = vault[slug];
    if (!p) return ``;
    
    return `
      <div class="aff-box" data-asin="${p.asin}">
        <img src="${p.localImage}" alt="${p.fullName}">
        <div class="aff-info">
          <h4>${p.fullName}</h4>
          <span class="aff-price">${p.price}</span>
          <a href="${p.affiliateUrl}" target="_blank" rel="nofollow" class="aff-btn">View on Amazon</a>
        </div>
      </div>
    `;
  });
}

You’re right. The index page works because it has a very specific design language, and my earlier version broke that language by introducing new visual ideas that weren’t already in the system. The strongest pages on this site feel editorial, restrained, and consistent, not “decorated.”

What index.html does well
The homepage is built around a few clear principles: a warm neutral palette, strong typographic hierarchy, large breathing room, and a small set of repeating card-like modules. It uses contrast carefully — serif headings for character, sans-serif body text for clarity, and green accents only to guide attention.

It also has a very disciplined layout rhythm. Sections are separated by generous padding, content is locked to a consistent max width, and components repeat in a predictable way, which makes the page feel calm and premium rather than busy.

Core design principles
Editorial restraint. The page avoids excess decoration and lets typography, spacing, and structure do the work. The result feels confident rather than flashy.

Consistent color system. It stays mostly within cream, ink, white, and a single green accent family. That limited palette makes the whole site feel unified and intentional.

Strong type hierarchy. The index page relies on large serif headlines, smaller uppercase labels, and lighter body copy. This makes it easy to scan while still feeling refined.

Clean spacing and alignment. Plenty of whitespace and a disciplined grid keep everything legible. Nothing feels crowded, and the eye always knows where to go next.

Card-based structure. Content is organized into clear panels, cards, and sections with consistent borders and rounded corners. That repetition creates a visual system, not a one-off design.

Selective emphasis. Buttons, badges, and accent text are used sparingly. Because attention is not overused, the important elements feel important.

Why it works so well
The homepage works because it feels like one coherent brand, not a collection of individual page decisions. Every element supports the same mood: calm, informed, trustworthy, and a little premium.

It also balances beauty with utility. The design is attractive, but it never fights the content; instead, it gives the content room to breathe and makes the structure easy to understand. That is why it feels polished rather than “designed.”

        <p>Begin with your city or county zoning office. You can usually find ordinances online, but a phone call is often faster. Ask specifically:</p>
        <ul>
          <li>Are beehives allowed in residential zones?</li>
          <li>How many hives are permitted per property?</li>
          <li>What setback distance is required from property lines?</li>
          <li>Do I need a permit or license?</li>
          <li>Are there seasonal restrictions?</li>
        </ul>
        <p>If you live in an HOA community, check your CC&amp;Rs (Covenants, Conditions &amp; Restrictions) and contact your HOA board directly. Some HOAs prohibit beekeeping entirely, while others have no rules against it. Getting written approval from your HOA before you start saves headaches later.</p>
        <h3>Common Zoning Requirements</h3>
        <p>Most municipalities that allow beekeeping require:</p>
        <ul>
          <li><strong>Setback distance:</strong> Usually 10-25 feet from property lines or neighboring structures</li>
          <li><strong>Hive limits:</strong> Often 2-4 hives per residential lot</li>
          <li><strong>Permits:</strong> Some cities require annual permits ($25-100); others have none</li>
          <li><strong>Registration:</strong> Many states require hive registration with the state apiary inspector</li>
        </ul>
        <p>Urban areas tend to have stricter rules, but many cities are loosening restrictions as urban beekeeping gains popularity. If your municipality doesn't have clear beekeeping ordinances, contact your city council or planning department about the process for getting approval.</p>
        <h3>State Registration and Inspection</h3>
        <p>Most states require you to register your hives with the state apiary inspector. This is usually free or low-cost and takes 10 minutes. Registration helps the state track bee health, disease, and swarm activity. Your state's Department of Agriculture website will have registration forms and requirements.</p>
        <h3>Liability Insurance</h3>
        <p>Check your homeowner's insurance policy. Some policies cover liability from bee stings; others don't. A few beekeeping associations offer group liability insurance (usually $100-200 annually). If you're in an urban area or have close neighbors, liability coverage is worth the investment.</p>
        <h3>What to Do If Your Area Has No Clear Rules</h3>
        <ul>
          <li><strong>Get written permission from your city.</strong> Contact the planning department and ask for written confirmation that beekeeping is allowed.</li>
          <li><strong>Start small and be discreet.</strong> Many beekeepers in gray-area zones keep 1-2 hives in a back corner, inform neighbors proactively, and never have problems.</li>
          <li><strong>Work with local beekeeping associations to create ordinances.</strong> Some communities are developing beekeeping rules specifically because local beekeepers advocated for them.</li>
        </ul>
        <p><strong>Bottom line:</strong> Do this research before you buy your first hive. A 10-minute phone call to your zoning office can save you from having to remove hives later.</p>
      </section>

      <section id="real-startup-costs">
        <h2>The Real Cost of Starting Beekeeping: Budget Breakdown</h2>
        <p>Beekeeping doesn't have to be expensive, but it's not free. Here's what you'll actually spend in year one, broken down by category.</p>
        <h3>Hive Setup: $300-500</h3>
        <p>Your hive is the foundation. You have two options:</p>
        <ul>
          <li><strong>Unassembled kit (budget option):</strong> $250-350. You'll assemble the boxes, frames, and bottom board yourself. Saves money but requires a few hours of work and basic tools.</li>
          <li><strong>Pre-assembled kit (convenience option):</strong> $400-500. Everything comes ready to use. Worth it if you're not handy or don't have time to assemble.</li>
        </ul>
        <p>A beginner kit typically includes one deep box (for brood), one medium box (for honey), frames, foundation, bottom board, inner cover, and outer cover. This is enough to start one hive. Most beginners start with one hive, though two is ideal for comparison and backup if one fails.</p>
        <p><a class="affiliate-link" href="AFFILIATEPLACEHOLDERbeginner-hive-kits">Beginner hive kits</a> from reputable suppliers range from budget-friendly to premium. Start with a mid-range option — you don't need the most expensive, but the cheapest kits often have quality issues.</p>
        <h3>Protective Gear: $100-200</h3>
        <p>You need to protect yourself from stings. Minimum gear:</p>
        <ul>
          <li><strong>Veil:</strong> $20-40. Protects your face and neck. Non-negotiable.</li>
          <li><strong>Gloves:</strong> $15-30. Leather or nitrile. You'll want multiple pairs.</li>
          <li><strong>Jacket or suit:</strong> $50-100. A light jacket with elastic cuffs is fine for beginners; full suits are overkill initially.</li>
          <li><strong>Boots:</strong> $30-50. Wear closed-toe shoes you don't mind getting stung through. Bees can crawl up pant legs, so tuck pants into socks or boots.</li>
        </ul>
        <p>Budget option: veil + gloves + light jacket = ~$100. Premium option: full suit + quality gloves + boots = ~$200. You can upgrade later as you learn your comfort level.</p>
        <p><a class="affiliate-link" href="AFFILIATEPLACEHOLDERbeginner-protective-gear">Beginner protective gear bundles</a> offer good value and usually include veil, gloves, and jacket together.</p>
        <h3>Essential Tools: $50-100</h3>
        <p>You need these:</p>
        <ul>
          <li><strong>Hive tool:</strong> $8-15. A flat metal tool for prying open boxes and scraping frames.</li>
          <li><strong>Smoker:</strong> $20-40. Burns fuel (wood chips, pellets) to calm bees during inspections.</li>
          <li><strong>Bee brush:</strong> $5-10. Gently brushes bees off frames.</li>
          <li><strong>Feeder:</strong> $10-20. For supplemental feeding in spring and fall.</li>
        </ul>
        <p><a class="affiliate-link" href="AFFILIATEPLACEHOLDERessential-tool-kits">Essential tool kits</a> bundle these items and cost $50-80, saving you money versus buying individually.</p>
        <p>Skip for now: uncapping knife, frame grip, queen excluder, honey extractor. You don't need these in year one.</p>
        <h3>Bees: $150-250</h3>
        <p>Your actual bees come in three forms:</p>
        <ul>
          <li><strong>Package bees:</strong> $120-150. A 3-pound box of loose bees with a queen. Most affordable, slower start, but reliable.</li>
          <li><strong>Nucleus colony (nuc):</strong> $150-250. A small established colony with frames, brood, and honey. Faster buildup, higher cost, limited availability.</li>
          <li><strong>Swarms:</strong> Free to $50. Unpredictable, requires experience. Not recommended for beginners.</li>
        </ul>
        <p>For beginners, packages or nucs are your best bet. Packages are cheaper; nucs get you ahead faster. Order in fall or early winter for spring delivery.</p>
        <h3>Feeding and Maintenance: $50-100 (Year One)</h3>
        <p>Bees need supplemental feeding in spring (before nectar flow) and fall (before winter). Budget:</p>
        <ul>
          <li><strong>Sugar for syrup:</strong> $20-30. You'll make 1:1 or 2:1 sugar-water syrup.</li>
          <li><strong>Pollen substitute (optional):</strong> $15-25. Helps in early spring.</li>
          <li><strong>Treatments for varroa mites:</strong> $20-40. Essential for hive health.</li>
        </ul>
        <h3>Year One Total: $650-1,150</h3>
        <p>Budget breakdown for one hive:</p>
        <ul>
          <li>Hive setup: $300-500</li>
          <li>Protective gear: $100-200</li>
          <li>Tools: $50-100</li>
          <li>Bees: $150-250</li>
          <li>Feeding and treatments: $50-100</li>
        </ul>
        <p><strong>Minimum (budget approach):</strong> ~$650. Unassembled kit, package bees, basic gear.</p>
        <p><strong>Comfortable (recommended):</strong> ~$900. Pre-assembled kit, nuc, decent gear.</p>
        <p><strong>Premium:</strong> ~$1,150+. Top-tier equipment, full suit, extra tools.</p>
        <h3>Ongoing Annual Costs (Year 2+): $200-400</h3>
        <p>After year one, costs drop significantly:</p>
        <ul>
          <li>Replacement frames and foundation: $30-50</li>
          <li>Varroa mite treatments: $30-50</li>
          <li>Sugar for feeding: $20-30</li>
          <li>Replacement bees (if a hive dies): $150-250</li>
          <li>Miscellaneous supplies: $20-50</li>
        </ul>
        <p>If you harvest honey, you'll recoup some costs. A strong hive produces 30-60 pounds of surplus honey annually, worth $200-400 at local market prices.</p>
        <p><strong>The bottom line:</strong> Beekeeping is affordable to start, but it's not free. Budget $700-900 for year one, then $200-400 annually. If you're on a tight budget, start with one hive, buy an unassembled kit, and use package bees.</p>

      <section id="time-commitment-by-season">
        <h2>Time Commitment: How Much Work Is Beekeeping Really?</h2>
        <p>Beekeeping time varies dramatically by season. Here's what to expect throughout the year.</p>
        <h3>Spring (March-May): 1-2 Hours Per Week</h3>
        <p>This is your busiest season. Bees are building up population, and you need to monitor closely.</p>
        <ul>
          <li><strong>Weekly inspections:</strong> 1-1.5 hours. Open the hive, check for brood patterns, look for disease, ensure the queen is laying.</li>
          <li><strong>Feeding:</strong> 30 minutes. Check feeders, refill with sugar syrup if needed.</li>
          <li><strong>Swarm prevention:</strong> 30 minutes. Add boxes before the hive gets too crowded.</li>
        </ul>
        <p>Total: 1.5-2 hours per week during peak spring activity.</p>
        <h3>Summer (June-August): 2-3 Hours Per Week</h3>
        <p>The hive is at full strength. Your focus shifts to honey production and disease management.</p>
        <ul>
          <li><strong>Bi-weekly inspections:</strong> 1-1.5 hours. You can inspect every 10-14 days instead of weekly once the hive is established.</li>
          <li><strong>Swarm prevention:</strong> 30 minutes. Continue monitoring for signs of swarming.</li>
          <li><strong>Pest and disease monitoring:</strong> 30 minutes. Watch for varroa mites, small hive beetles, or other issues.</li>
          <li><strong>Honey super management:</strong> 30 minutes. Add boxes as bees fill frames with honey.</li>
        </ul>
        <p>Total: 2-3 hours per week during summer.</p>
        <h3>Fall (September-October): 3-4 Hours Total</h3>
        <p>This is harvest time and prep for winter. It's concentrated work over a few weeks, not ongoing.</p>
        <ul>
          <li><strong>Honey harvest:</strong> 2-3 hours. Remove honey supers, extract honey, clean equipment.</li>
          <li><strong>Hive assessment:</strong> 1 hour. Check that the hive has enough honey stores for winter (40-60 pounds).</li>
          <li><strong>Feeding for winter:</strong> 30 minutes. If stores are low, feed sugar syrup or fondant.</li>
          <li><strong>Varroa treatment:</strong> 30 minutes. Apply mite treatments before winter.</li>
        </ul>
        <p>Total: 3-4 hours spread over September and October, not weekly.</p>
        <h3>Winter (November-February): Minimal, ~30 Minutes Per Month</h3>
        <p>Bees cluster inside and don't need much from you.</p>
        <ul>
          <li><strong>Monthly checks:</strong> 15-30 minutes. Peek at the entrance to confirm the hive is alive. Don't open it — cold air can kill bees.</li>
          <li><strong>Feeding (if needed):</strong> 15 minutes. In late winter, if stores are low, add fondant or sugar cakes.</li>
          <li><strong>Equipment maintenance:</strong> 1-2 hours total over the season. Clean and repair gear, plan for spring.</li>
        </ul>
        <p>Total: 30 minutes per month, plus 1-2 hours of equipment work.</p>
        <h3>Year One Learning Curve</h3>
        <p>In your first year, expect to spend 20-30% more time than these estimates. You'll be learning, second-guessing yourself, and consulting guides frequently. This is normal. By year two, you'll work faster and more confidently.</p>
        <h3>Two Hives vs. One Hive</h3>
        <p>If you keep two hives, add about 30-40% to your time commitment. You're not doubling your work — you're doing similar tasks twice, but some tasks (like learning, planning, equipment maintenance) don't scale linearly.</p>
        <h3>Is It Worth the Time?</h3>
        <p>Most beekeepers find the time commitment reasonable, especially once they're past year one. The work is seasonal, not year-round. Summer requires more attention, but winter is almost hands-off. If you enjoy outdoor work and problem-solving, you'll find the time worthwhile. If you're looking for a passive income stream, beekeeping isn't it — at least not in year one.</p>
      </section>

      <section id="space-requirements">
        <h2>Space Requirements: Urban, Suburban, and Rural Beekeeping</h2>
        <p>Hives don't need much space, but placement matters. Here's what works in different settings.</p>
        <h3>Urban Beekeeping (Apartments, Small Yards, Rooftops)</h3>
        <p>You can keep bees in the city, but space is tight.</p>
        <ul>
          <li><strong>Minimum space:</strong> A rooftop, small fenced yard, or balcony with at least 50 square feet of dedicated space.</li>
          <li><strong>Hive placement:</strong> Put hives in a back corner, elevated if possible, facing away from neighbors' windows and patios.</li>
          <li><strong>Flight path:</strong> Bees fly in a relatively straight line from the hive. Orient the entrance toward open space (alley, park, street) rather than toward neighbors' yards.</li>
          <li><strong>Screening:</strong> A 6-foot fence or hedge between your hive and neighbors' property helps redirect bee flight paths upward.</li>
          <li><strong>Hive number:</strong> Most urban ordinances allow 1-2 hives. Check your local rules.</li>
        </ul>
        <p>Urban beekeeping works best with engaged neighbors who understand what you're doing. Proactive communication (covered in the next section) is essential.</p>
        <h3>Suburban Beekeeping (Typical Backyard)</h3>
        <p>This is the sweet spot for most beginners.</p>
        <ul>
          <li><strong>Minimum space:</strong> A quarter-acre lot with a dedicated hive area.</li>
          <li><strong>Ideal placement:</strong> Back corner of the yard, 10-25 feet from property lines (check your local setback requirements). Morning sun, afternoon shade, wind protection.</li>
          <li><strong>Flight path:</strong> Orient hive entrance toward the back of your property or toward open space. Avoid pointing toward neighbors' homes or high-traffic areas.</li>
          <li><strong>Screening:</strong> A 4-6 foot fence or dense hedge is ideal. Bees flying over obstacles climb higher and are less likely to interact with neighbors.</li>
          <li><strong>Hive number:</strong> 2-4 hives is typical for suburban lots. Space them 3-4 feet apart to prevent drifting.</li>
        </ul>
        <p>Suburban yards usually offer enough space for 2-3 hives without crowding. If you have room, two hives are better than one — they provide backup if one fails and give you comparison data.</p>
        <h3>Rural Beekeeping (Acreage)</h3>
        <p>Rural settings offer the most flexibility.</p>
        <ul>
          <li><strong>Minimum space:</strong> No real minimum. You can space hives further apart and have more freedom.</li>
          <li><strong>Ideal placement:</strong> 50+ feet from your home (for noise and smell), facing open fields or woods. Morning sun, afternoon shade.</li>
          <li><strong>Predators:</strong> Rural areas have bears, raccoons, and other wildlife. Plan for electric fencing or robust hive stands.</li>
          <li><strong>Hive number:</strong> 5-20+ hives is common for rural beekeepers. Space them 4-6 feet apart.</li>
        </ul>
        <p>Rural beekeeping offers more space but introduces predator management as a concern. If you're in bear country, electric fencing is essential.</p>
        <h3>Key Placement Principles (All Settings)</h3>
        <ul>
          <li><strong>Morning sun:</strong> Bees need warmth to get active. East-facing entrances catch morning sun.</li>
          <li><strong>Afternoon shade:</strong> In hot climates, afternoon shade prevents overheating.</li>
          <li><strong>Wind protection:</strong> Hives in windy spots stress bees. Use a fence, hedge, or natural windbreak.</li>
          <li><strong>Water access:</strong> Bees need water. If there's no natural source nearby, provide a shallow dish with pebbles or straw.</li>
          <li><strong>Elevation:</strong> Slightly elevated hives (on a stand or platform) improve drainage and reduce pest access.</li>
          <li><strong>Distance from activity:</strong> Avoid placing hives near your patio, kids' play area, or dog run. Bees are defensive when disturbed.</li>
        </ul>
        <h3>Hive Stand Setup</h3>
        <p>Hives should sit on a stand 12-18 inches off the ground. This improves ventilation, prevents ground moisture, and deters pests. A simple wooden stand or concrete blocks work fine. Make sure the stand is level and sturdy enough to hold 100+ pounds.</p>
        <h3>Space Planning Tip</h3>
        <p>If you're considering multiple hives, space them at least 3-4 feet apart. This prevents bees from one hive drifting into another and reduces disease transmission if one hive gets sick.</p>
      </section>

      <section id="talking-to-neighbors">
        <h2>How to Talk to Your Neighbors About Keeping Bees</h2>
        <p>This conversation can make or break your beekeeping experience. Do it right, and you'll have supportive neighbors. Do it wrong, and you'll face complaints, pressure to remove hives, or worse. Here's how to approach it.</p>
        <h3>When to Tell Your Neighbors</h3>
        <p><strong>Tell them before you install hives.</strong> Don't surprise them. Proactive communication builds trust; discovering bees in your yard after the fact breeds resentment. Ideally, mention it 2-4 weeks before you plan to install hives.</p>

/**
 * manifest-factory.js
 * Generates bulk page entries for apiary-manifest.json
 */

const fs = require('fs');
const path = require('path');

const MANIFEST_PATH = path.join(__dirname, 'apiary-manifest.json');
const manifest = JSON.parse(fs.readFileSync(MANIFEST_PATH, 'utf8'));

// ── Templates for Bulk Generation ─────────────────────────────────────────────

const US_STATES = ['Maryland', 'Virginia', 'Pennsylvania', 'New York', 'Texas']; // Expand this to 50

/**
 * Generates localized "Beekeeping in [State]" pages
 */
function generateLocalGuides() {
  return US_STATES.map(state => ({
    slug: `beekeeping-in-${state.toLowerCase().replace(/ /g, '-')}`,
    title: `Beekeeping in ${state}: A Complete Regional Guide`,
    type: "guide",
    pillar: "local",
    priority: 3,
    searchIntent: "informational",
    notes: `Focus on ${state} specific climate, native nectar flows, and state-level registration laws. Link to ${state} beekeeping associations.`
  }));
}

/**
 * Generates technical "Comparison" pages
 */
const COMPARISONS = [
  ['Trek Marlin 7', 'Trek Verve 2'], // Example of your interests influencing tech comparisons
  ['Langstroth', 'Flow Hive'],
  ['Plastic Frames', 'Beeswax Foundation']
];

function generateComparisons() {
  return COMPARISONS.map(([itemA, itemB]) => ({
    slug: `${itemA.toLowerCase().replace(/ /g, '-')}-vs-${itemB.toLowerCase().replace(/ /g, '-')}`,
    title: `${itemA} vs ${itemB}: Which is Better for Your Hive?`,
    type: "comparison",
    pillar: "shop",
    priority: 2,
    searchIntent: "research",
    notes: `Head-to-head comparison of ${itemA} and ${itemB}. Focus on durability, price, and ease of use for beginners.`
  }));
}

// ── Execution ────────────────────────────────────────────────────────────────

const newPages = [
  ...generateLocalGuides(),
  ...generateComparisons()
];

// Append to manifest while avoiding duplicates
newPages.forEach(newPage => {
  if (!manifest.pages.find(p => p.slug === newPage.slug)) {
    manifest.pages.push(newPage);
  }
});

fs.writeFileSync(MANIFEST_PATH, JSON.stringify(manifest, null, 2), 'utf8');
console.log(`Added ${newPages.length} pages to the manifest.`);

---

## Pipeline Architecture

Two-pass batch generation driven by apiary-manifest.json:

- Pass 1 — Haiku/Sonnet architects each page
  - Input: title, type, pillar, search intent, notes from manifest
  - Output: structured JSON outline saved to data/outlines/<slug>.json
- Pass 2 — Haiku implements the outline
  - Input: outline JSON + HTML template in renderPage()
  - Output: complete HTML page written to root folder

Commands (run from apiary/ folder):
- node pipeline-apiary-batch.js              (priority 1 pages — default)
- node pipeline-apiary-batch.js --all        (all 30 pages)
- node pipeline-apiary-batch.js --slug X     (single page)
- node pipeline-apiary-batch.js --pass 2     (re-render from existing outlines)
- node pipeline-apiary-batch.js --dry        (preview only, no API calls)

## Design System
- Based exactly on Apex Search Co. (style-examples/apex-search.html)
- Forest green accent: #2d5a3d / #4a8c60
- Cream background: #f7f5f0
- Cormorant Garamond serif headings (lightweight, editorial)
- DM Sans body (clean, minimal)
- Eyebrow line pattern on all section headers
- Index page: fixed nav with backdrop blur, hero grid, stats card
- Inner pages: no blur, sticky sidebar, Cormorant heading, green CTA block
