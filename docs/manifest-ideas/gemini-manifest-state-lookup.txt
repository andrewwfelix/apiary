const STATE_DATA = {
  "Maryland": { zone: "7a-8a", honey: "Tulip Poplar, Clover", law: "Registration required with MDA" },
  "Texas": { zone: "6b-10a", honey: "Mesquite, Tallow", law: "Managed by Texas A&M Apiary Inspection Service" },

};
2. The Generator Logic
This function automates the creation of the pages array entries. It transforms raw data into the "Instruction Set" that Pass 1 (Claude Sonnet) will use to build the outline.

JavaScript
function generateLocalPages() {
  return Object.entries(STATE_DATA).map(([state, data]) => {
    return {
      slug: `beekeeping-in-${state.toLowerCase().replace(/ /g, '-')}`,
      title: `Beekeeping in ${state}: A Regional Guide`,
      type: "guide",
      pillar: "local",
      priority: 3,
      // This is the CRITICAL logic: feeding the AI the "Hard Facts" 
      notes: `Focus on ${state} specifics. 
              Climate: USDA Zones ${data.zone}. 
              Primary Nectar: ${data.honey}. 
              Legal: Mention ${data.law}. 
              Link to ${state} Beekeeping Association.`
    };
  });
}
3. Why This Works
The AI is no longer "Fishing": In Pass 1, the AI sees: "Climate: USDA Zones 7a-8a. Primary Nectar: Tulip Poplar." It doesn't have to search its memory; it just has to explain how a beekeeper in Zone 7 should prep for winter.

Consistency: Every state page follows the same high-quality structure (Intro -> Climate -> Flora -> Legal -> Local Clubs), which is excellent for SEO "Topical Authority."

AEC Professionalism: This mirrors a "Standard Operating Procedure." You are providing the specifications, and the AI is acting as the technical writer.
